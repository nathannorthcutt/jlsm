## `.map` file format (JSON)

Every feature being migrated has a `.map` file. The map is the authoritative record of where each requirement lands.

```json
{
  "source_spec": "F13",
  "source_path": ".spec/domains/engine/F13-jlsm-schema.md",
  "source_state": "APPROVED",
  "total_requirements": 59,
  "requirements": {
    "R1": { "new_spec": "schema.schema-construction", "new_rn": "R1", "new_path": ".spec/domains/schema/schema-construction.md" },
    "R2": { "new_spec": "schema.schema-construction", "new_rn": "R2", "new_path": ".spec/domains/schema/schema-construction.md" },
    "R13": { "new_spec": "schema.schema-field-access", "new_rn": "R1", "new_path": ".spec/domains/schema/schema-field-access.md" }
  },
  "destinations": [
    {
      "new_spec": "schema.schema-construction",
      "new_path": ".spec/domains/schema/schema-construction.md",
      "domains": ["schema"],
      "source_requirements": ["R1", "R2", "..."]
    }
  ]
}
```

## Invariants (enforced by `01-validate-maps.sh`)

1. Every `R*` key from the source spec appears exactly once in `requirements`.
2. `destinations[].source_requirements` arrays collectively partition the `requirements` key set (no overlap, no missing).
3. Within each destination, `new_rn` values form a consecutive sequence starting at R1 (no gaps, no duplicates).
4. Every `requirements[Rn].new_spec` matches some `destinations[i].new_spec`.
5. `source_state` matches the `state` field of the source spec's frontmatter (sanity check).

## Feature classification

**7 features distribute across multiple destinations** — hand-crafted:
- F02 (block-compression) → compression + sstable
- F03 (encrypt-memory-data) → 4 encryption/primitives-* specs
- F10 (table-indices-and-queries) → 8 query/* specs
- F13 (jlsm-schema) → 5 schema/schema-* specs
- F14 (jlsm-document) → 4 schema/document-* specs
- F17 (wal-compression-codec-api) → compression + wal + sstable (some reqs merge into F02's compression/codec-contract.md)
- F42 (wal-encryption) → wal + encryption (primitive reqs to encryption/, application reqs to wal/)

Within each hand-crafted map, RNs renumber consecutively from R1 within each destination spec.
