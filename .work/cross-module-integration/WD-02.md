## Summary

Wire LsmVectorIndex (IvfFlat/Hnsw) from jlsm-vector through the jlsm-core
interface boundary so that VectorFieldIndex delegates to a real implementation.
Currently mutations are silent no-ops and lookup throws
UnsupportedOperationException — tables with VECTOR indices accept writes
that are never actually indexed, which is a data integrity hazard.

## Acceptance Criteria

- F10.R85 satisfied: onInsert propagates vector data to LsmVectorIndex
- F10.R86 satisfied: onUpdate propagates vector changes to LsmVectorIndex
- F10.R87 satisfied: onDelete removes vector from LsmVectorIndex
- F10.R88 satisfied: lookup returns nearest-neighbor results via
  LsmVectorIndex
- F10.R89 satisfied: supports() returns true for VECTOR predicates
- F10.R90 satisfied: close() cleans up LsmVectorIndex resources
- F10.R6 (partial): tables with VECTOR indices index writes instead of
  silently dropping them
- Obligation OBL-F10-vector resolved in _obligations.json

## Implementation Notes

**Module boundary crossing:**
- jlsm-vector exports LsmVectorIndex (IvfFlat and Hnsw variants)
- jlsm-core defines FieldIndex interface (VectorFieldIndex is the stub)
- VectorFieldIndex must delegate to LsmVectorIndex via constructor
  injection or factory pattern
- IndexRegistry needs to resolve the concrete vector implementation at
  table creation time

**Critical safety concern:**
- Current silent no-op mutations are worse than throwing — data appears
  to be indexed but isn't. Priority: make onInsert/onUpdate/onDelete
  either work or throw, never silently drop.

**Key files (from obligation description):**
- VectorFieldIndex.java in jlsm-core — the stub to replace
- IndexRegistry.java — already validates VectorType field constraints
- LsmVectorIndex in jlsm-vector — the real implementation

**Tests needed:**
- End-to-end: create table with VECTOR index, insert vectors, nearest-
  neighbor query
- Mutation lifecycle: insert → update → delete → verify index state
- Field type validation: VECTOR on non-VectorType field rejected
- Index algorithm selection: IvfFlat vs Hnsw based on configuration
- Close lifecycle: index resources released on table shutdown
