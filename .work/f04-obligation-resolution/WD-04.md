## Summary

Implement full RAPID consensus to replace the current unilateral protocol.
The current Rapid-inspired implementation has each node apply suspicions
independently. The spec requires multi-process cut detection with quorum-
based consensus rounds, observer-agreement broadcast, self-refutation via
incarnation bump, and an expander-graph overlay for O(log N) monitoring.

## Acceptance Criteria

- F04.R34 satisfied: suspicion about a member requires agreement from
  a quorum (consensusQuorumPercent, default 75%) of observers before
  the view change is applied
- F04.R35 satisfied: monitoring is distributed across an expander-graph
  overlay with configurable degree; each node monitors O(degree) peers
  instead of all ALIVE members; graph is maintained under churn
- F04.R36 satisfied: consensus rounds have a bounded timeout; if quorum
  is not reached within the timeout, the round is abandoned and the
  member remains in its current state
- F04.R37 satisfied: observer-agreement is broadcast before suspicion
  application; observers share their suspicion evidence before the cut
  is committed
- F04.R38 satisfied: a live member receiving a suspicion about itself
  refutes by broadcasting a higher-incarnation alive announcement; the
  refutation cancels the pending consensus round
- Obligations OBL-F04-R34-38-consensus and OBL-F04-R35-expander-graph
  resolved in _obligations.json

## Implementation Notes

**Consensus protocol:**
- Add ConsensusRound tracking: proposed member, observer set, votes
  received, timeout deadline
- protocolTick: when phi exceeds threshold for a monitored peer, start
  a consensus round instead of immediately applying SUSPECTED
- Observer-agreement: broadcast SUSPICION_PROPOSAL to the peer's observer
  set; collect SUSPICION_VOTE responses
- Quorum check: if votes >= ceil(observers * quorumPercent / 100),
  apply the view change; otherwise abandon on timeout
- Self-refutation: if a node receives SUSPICION_PROPOSAL about itself,
  increment incarnation and broadcast ALIVE with new incarnation

**Expander graph:**
- Build a k-regular random graph over ALIVE members (k = configurable
  degree, default log2(clusterSize))
- Each node monitors only its graph neighbors
- Rebuild graph incrementally on join/leave (add/remove vertex, rewire
  affected edges)
- Tests: graph connectivity under sequential node failures, monitoring
  load distribution
