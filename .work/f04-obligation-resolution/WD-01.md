## Summary

Fix two foundational protocol-layer issues that other WDs depend on.
R39 requires a dedicated listener executor so slow/throwing listeners
cannot block protocol message processing. R53 requires monotonic clock
usage so backward wall-clock adjustments cannot extend grace windows.

## Acceptance Criteria

- F04.R39 satisfied: listener notification is fully asynchronous via a
  dedicated executor; a slow listener does not delay protocol tick
  processing; a throwing listener does not crash the protocol thread
- F04.R53 satisfied: GracePeriodManager uses monotonic time internally;
  backward clock adjustments do not extend grace windows (existing
  adversarial test passes without the clock-backward-extends-grace
  behavior)
- Obligations OBL-F04-R39-async-listeners and OBL-F04-R53-monotonic-clock
  resolved in _obligations.json

## Implementation Notes

**R39 — Async listeners:**
- Add a bounded single-thread executor for listener dispatch
- Listener callbacks invoked via executor.submit(), not direct call
- Drop or backpressure policy on overflow (design choice during implementation)
- Tests: slow listener (Thread.sleep in callback), throwing listener
  (RuntimeException), listener ordering preservation

**R53 — Monotonic clock:**
- Implement MonotonicClock wrapping System.nanoTime() with a fixed epoch
- Migrate GracePeriodManager to use MonotonicClock internally
- Existing Clock injection remains for testability, but default changes
  to monotonic
- Tests: clock backward jump no longer extends grace period (flip existing
  adversarial test from demonstrating the bug to asserting the fix)

**Spec reference:** F04 requirements R39, R53
